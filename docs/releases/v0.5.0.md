# macVR 0.5.0

Release date: 2026-03-23

## Summary

`0.5.0` connects the transport layer to the experimental OpenXR runtime more directly. The bundled runtime and standalone host now persist the latest validated head pose into a shared binary tracking-state file, and the OpenXR shim reads that file when answering `xrLocateSpace` and `xrLocateViews`.

## Highlights

- New binary tracking-state handoff shared between the Swift transport/runtime layer and the C OpenXR runtime shim
- New `--tracking-state-path` option for `macvr-runtime` and `macvr-host`
- OpenXR pose queries now use live transport-driven head pose when available
- Quaternion normalization at the tracking-state boundary for more stable pose math
- Updated synthetic pose generation in `macvr-client` and `macvr-viewer` to emit proper yaw quaternions
- Release packaging now includes the raw `macvr-control-center` executable under `bin/`

## Included Runtime Surface

- Experimental OpenXR runtime shim: included
- Bundled macOS runtime service: included
- Graphical macOS control center: included
- Graphical macOS viewer/receiver: included
- Versioned macOS `.app` bundles: included
- Native live display capture sender: included
- Binary head-pose handoff for loader-driven OpenXR pose queries: included

## Current Limitations

- The included OpenXR runtime is still experimental and headless-first. It does not yet provide production graphics API or swapchain support.
- The sender/runtime/viewer loop now feeds head pose into the OpenXR shim, but it is still not a Quest-ready compositor/runtime with full controller tracking and headset presentation.
- SteamVR, Wine, Proton, ALVR, Virtual Desktop, and Apple Game Porting Toolkit remain optional external tools. They are not bundled or redistributed by this repository.
- The packaged `.app` bundles are ad-hoc signed for local distribution and are not notarized with an Apple Developer ID certificate.
- Live display capture still depends on macOS Screen Recording permission for the launching terminal or packaged app.
