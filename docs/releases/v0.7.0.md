# macVR 0.7.0

Release date: 2026-04-08

## Summary

`0.7.0` adds an ALVR-style trusted-client workflow so runtime control sessions can be restricted to explicitly approved clients. The trust system is persistent, exposed in both CLI and control-center GUI, and includes status observability for denied untrusted handshakes.

## Highlights

- Added `TrustedClientStore` in `MacVRRuntimeCore` for persistent JSON allowlist storage.
- Added runtime trust enforcement through `RuntimeTrustPolicy` and host authorization callbacks.
- Added runtime CLI trust controls:
  - `--require-trusted-clients`
  - `--no-auto-trust-loopback`
  - `--trusted-clients-path`
  - `--trust-client <client@host>`
  - `--untrust-client <client@host>`
  - `--list-trusted-clients`
- Added control-center trusted-client management UI with hover tooltips on all controls.
- Added runtime integration coverage for trusted-client persistence and strict-trust policy behavior.

## Included Runtime Surface

- Experimental OpenXR runtime shim: included
- Bundled macOS runtime service: included
- Graphical macOS control center: included
- Graphical macOS viewer/receiver: included
- Versioned macOS `.app` bundles with icon resource: included
- Runtime discovery protocol and viewer discovery GUI: included
- Trusted-client policy and persistent allowlist: included

## Current Limitations

- The included OpenXR runtime remains experimental and headless-first. It does not provide production graphics API or swapchain support.
- Discovery and trust workflows are now integrated, but full ALVR/SteamVR feature parity is still incomplete (no full compositor/controller stack equivalent yet).
- SteamVR, Wine, Proton, ALVR, Virtual Desktop, and Apple Game Porting Toolkit remain optional external tools and are not bundled by this repository.
- Packaged `.app` bundles are ad-hoc signed and not notarized with an Apple Developer ID certificate.
