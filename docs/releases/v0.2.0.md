# macVR 0.2.0

Release date: 2026-03-15

## Summary

`0.2.0` adds the missing native runtime surface that earlier release notes called out as absent. The project now includes a bundled runtime service, an experimental OpenXR runtime shim, and a graphical macOS control center.

## Highlights

- Bundled `macvr-runtime` service with direct localhost JPEG ingest plus bridge-protocol ingest
- Experimental `MacVROpenXRRuntime` dylib that negotiates with the OpenXR loader and supports headless session flow
- New `macvr-control-center` SwiftUI app with hover tooltips on every interactive control
- Manifest generation support for the OpenXR runtime shim
- Runtime and OpenXR integration tests covering the new surface area
- Versioned release packaging that ships a macOS `.app` bundle, CLI binaries, dylib, zip archive, and checksum file

## Verification

- `swift build --disable-sandbox --scratch-path /tmp/macvr-swiftpm-build`
- `swift test --disable-sandbox --scratch-path /tmp/macvr-swiftpm-test`
- `macvr-runtime --write-openxr-manifest <path> --manifest-only`
- `nm -gU libMacVROpenXRRuntime.dylib`
- End-to-end smoke: `macvr-runtime` + `macvr-client` + `macvr-jpeg-sender`
- Control-center launch sanity check: start `macvr-control-center`, keep it alive briefly, then stop it cleanly
- Release packaging check: `scripts/release/build-release.sh`

## Included Runtime Surface

- Experimental OpenXR runtime shim: included
- Bundled macOS runtime service: included
- Graphical macOS control center: included
- Versioned macOS `.app` bundle: included

## Current Limitations

- The included OpenXR runtime is experimental and headless-first. It does not yet provide production graphics API or swapchain support.
- SteamVR, Wine, Proton, ALVR, Virtual Desktop, and Apple Game Porting Toolkit remain optional external tools. They are not bundled or redistributed by this repository.
- The packaged control center `.app` is ad-hoc signed for local distribution and is not notarized with an Apple Developer ID certificate.
