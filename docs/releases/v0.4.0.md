# macVR 0.4.0

Release date: 2026-03-23

## Summary

`0.4.0` turns the existing transport stack into a real macOS sender/runtime/receiver loop. The new `macvr-viewer` app negotiates sessions with the host or bundled runtime, receives UDP frame traffic, decodes JPEG payloads, and renders a stereo preview with live connection and throughput metrics.

## Highlights

- New `macvr-viewer` SwiftUI receiver with hover tooltips on every interactive control
- New `macvr-viewer --headless` mode for deterministic automated transport verification
- New packaged `macVR Viewer.app` alongside `macVR Control Center.app`
- Control center can now launch the packaged viewer directly from a release bundle
- Repository cleaned of reviewed-but-unintegrated renderer, audio, controller, and tracking experiments
- Custom `xr_metal.h` experiment removed from the OpenXR header surface

## Included Runtime Surface

- Experimental OpenXR runtime shim: included
- Bundled macOS runtime service: included
- Graphical macOS control center: included
- Graphical macOS viewer/receiver: included
- Versioned macOS `.app` bundles: included
- Native live display capture sender: included

## Current Limitations

- The included OpenXR runtime is experimental and headless-first. It does not yet provide production graphics API or swapchain support.
- The sender/runtime/viewer loop validates native macOS capture, transport, and stereo preview, but it is not yet a Quest-ready headset compositor with tracked controller input.
- SteamVR, Wine, Proton, ALVR, Virtual Desktop, and Apple Game Porting Toolkit remain optional external tools. They are not bundled or redistributed by this repository.
- The packaged `.app` bundles are ad-hoc signed for local distribution and are not notarized with an Apple Developer ID certificate.
- Live display capture still depends on macOS Screen Recording permission for the launching terminal or packaged app.
