# macVR 0.3.0

Release date: 2026-03-15

## Summary

`0.3.0` adds a native live display capture sender for macOS. The new `macvr-capture-sender` executable captures the desktop, scales and JPEG-encodes it, then feeds it directly into `macvr-runtime` or `macvr-bridge-sim` over the existing localhost TCP seam.

## Highlights

- New `macvr-capture-sender` executable for live macOS desktop capture
- Shared `DisplayCapture` helper used by both `display-jpeg` host mode and the new sender
- Control-center copy command now targets the live capture workflow by default
- Release bundle now ships `macvr-capture-sender` alongside the runtime, GUI, and existing CLI tools

## Included Runtime Surface

- Experimental OpenXR runtime shim: included
- Bundled macOS runtime service: included
- Graphical macOS control center: included
- Versioned macOS `.app` bundle: included
- Native live display capture sender: included

## Current Limitations

- The included OpenXR runtime is experimental and headless-first. It does not yet provide production graphics API or swapchain support.
- SteamVR, Wine, Proton, ALVR, Virtual Desktop, and Apple Game Porting Toolkit remain optional external tools. They are not bundled or redistributed by this repository.
- The packaged control center `.app` is ad-hoc signed for local distribution and is not notarized with an Apple Developer ID certificate.
- Live display capture still depends on macOS Screen Recording permission for the launching terminal or packaged app.
