# macVR 0.6.0

Release date: 2026-04-07

## Summary

`0.6.0` adds ALVR-style runtime discovery on macOS and ships a proper app icon in packaged GUI bundles. The runtime now advertises itself over UDP, and the viewer can discover and apply available runtimes from the GUI before opening a control-channel session.

## Highlights

- Added UDP runtime discovery protocol messages in `MacVRProtocol`.
- Added runtime discovery listener in `macvr-runtime`:
  - `--discovery-port`
  - `--server-name`
- Added viewer-side discovery workflow in GUI:
  - `Discovery Port` field
  - `Discover Runtimes` action
  - `Discovered Runtimes` list with `Use` buttons
- Added packaged app icon support via `assets/macvr.icns` and release packager Info.plist icon wiring.
- Moved incomplete runtime-core tracking plugin experiments to `experimental/runtime-core/` so default builds remain stable.

## Included Runtime Surface

- Experimental OpenXR runtime shim: included
- Bundled macOS runtime service: included
- Graphical macOS control center: included
- Graphical macOS viewer/receiver: included
- Versioned macOS `.app` bundles with icon resource: included
- Native live display capture sender: included
- Binary head-pose handoff for OpenXR pose queries: included

## Current Limitations

- The included OpenXR runtime remains experimental and headless-first. It still does not provide production graphics API or swapchain support.
- Discovery is ALVR-style for finding runtimes, but full ALVR/SteamVR feature parity is not complete yet (no full compositor/controller stack equivalent yet).
- SteamVR, Wine, Proton, ALVR, Virtual Desktop, and Apple Game Porting Toolkit remain optional external tools and are not bundled by this repository.
- Packaged `.app` bundles are ad-hoc signed and not notarized with an Apple Developer ID certificate.
